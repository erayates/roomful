# Support

## Where to Ask for Help

### 1) Q&A and Usage Discussions

Use GitHub Discussions for design questions, integration help, and open-ended guidance:

- <https://github.com/erayates/cahoots/discussions>

### 2) Confirmed Bugs

Use GitHub Issues for reproducible defects:

- <https://github.com/erayates/cahoots/issues>

Please use issue templates and include complete reproduction context.

### 3) Security Concerns

Use private vulnerability reporting only:

- <https://github.com/erayates/cahoots/security>

## Triage Expectations

Maintainers aim to:

- Label new issues/discussions during routine review windows
- Request missing details when reproduction context is incomplete
- Close stale issues when no reproduction or follow-up is provided

## What to Include in Support Requests

- Runtime environment (browser, Node version, OS)
- Package and version (or commit SHA)
- Minimal reproducible steps
- Expected behavior
- Actual behavior
- Logs or traces

## Scope of Maintainer Support

- Best effort for community support
- No guaranteed SLA
- Prioritization based on severity and project roadmap
